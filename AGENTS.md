# AGENTS.md — RustyNotes

Tauri 2.0 + SvelteKit (SPA) + TypeScript frontend / Rust backend. Obsidian-like note app.

## Commands

```bash
yarn tauri dev      # dev server + webview (runs yarn dev first)
yarn tauri build    # production build
yarn check          # type-check: svelte-kit sync && svelte-check
yarn check:watch    # same as check but in watch mode
```

No linter, no formatter, no test runner. `yarn check` is the only verification. `TAURI_DEV_HOST` env var enables network access (mobile dev) and HMR on port 1421.

## Rust crate naming quirk (Windows)

The Cargo.toml lib is named `rustynotes_lib` because cargo on Windows barfs when lib and bin have the same name. `main.rs` calls `rustynotes_lib::run()`. If an agent writes `rustynotes::run()`, it will fail.

## SvelteKit in SPA mode

- Uses `@sveltejs/adapter-static` with `fallback: "index.html"`.
- SSR is disabled: `src/routes/+layout.ts` exports `export const ssr = false`.
- Vite dev port is **1420** (fixed, strict); HMR on 1421 only when `TAURI_DEV_HOST` is set.
- Vite ignores `src-tauri/` in watch mode.
- Frontend build output goes to `../build` (wired in `tauri.conf.json`).

## Tauri 2 capabilities / permissions

Permissions are in `src-tauri/capabilities/default.json`. Currently `core:default` + `opener:default` + `dialog:default`. When adding new capabilities (FS ops, dialogs, etc.), Tauri 2 requires explicit permissions in this file — not just code changes.

Plugins registered in `lib.rs`: `tauri_plugin_opener`, `tauri_plugin_dialog`. If you add a new Tauri plugin, register it in both `lib.rs` and `Cargo.toml`, then add its permission to `capabilities/default.json`.

`tsconfig.json` extends `.svelte-kit/tsconfig.json` (generated). Running `tsc` directly will fail unless `svelte-kit sync` has run first. Use `yarn check` (which chains both) instead.

## Architecture

Single-crate, single-package repo (not a monorepo). **Most `docs/TASKS.md` tasks are already implemented** (T1–T15, T17–T18). Check actual files before reimplementing.

|---------|------|
| Frontend (SvelteKit) | `src/` |
| Backend (Rust/Tauri) | `src-tauri/src/` |

### Rust modules (`src-tauri/src/`)

- `notebook.rs` — create/open/list/close notebooks, global settings persisted to `~/.config/rustynotes/` (or `%APPDATA%/rustynotes/` on Windows)
- `notes.rs` — CRUD for `.md` files, tag extraction from content, tree-building
- `search.rs` — full-text search (regex + walkdir), uses `ignore` crate for `.gitignore` exclusions
- `tags.rs` — tag extraction across all notes, deduplication per file, code-block exclusion
- `fs_watcher.rs` — debounced (100ms) FS watcher using `notify` crate, emits `note-created`/`note-modified`/`note-deleted` events to frontend
- `state.rs` — `AppState` wrapped in `std::sync::Mutex` (not async RwLock)

### Frontend key paths

- `src/lib/commands.ts` — typed `invoke()` wrappers for all Tauri commands + `buildFileTree()` utility
- `src/lib/events.ts` — frontend listener for FS watcher events, refreshes the file tree
- `src/lib/editor-engine/` — CodeMirror 6 setup, wiki-link support, themes, editor modes, `codemirror-live-markdown` package for live preview, KaTeX math rendering (`math-live.ts`), lazy-reading-view
- `src/lib/stores/` — Svelte 5 runes stores for notes, search, settings, tabs, UI, notebook
- `src/lib/plugins/` — plugin type definitions and loader

## FS watcher behavior

- Uses `notify` crate (`Config::default().with_poll_interval(Duration::from_secs(1))`) — polling mode for cross-platform reliability.
- Only watches `.md` files; ignores other changes.
- Debounce window is 100ms.
- Emits `note-created`, `note-modified`, `note-deleted` events — frontend listens in `src/lib/events.ts`.

## State management

- Rust side: `Mutex<AppState>` (type alias `ManagedState`). Synchronous, not async.
- Frontend side: Svelte 5 runes (`.svelte.ts` files) using `$state`, `$derived`, `$effect`.

## Generated / git-ignored paths

- `.svelte-kit/` — generated by `svelte-kit sync` (part of `yarn check`). Required before type-checking.
- `src-tauri/target/` — Rust build artifacts (gitignored in `src-tauri/.gitignore`).
- `src-tauri/gen/schemas` — generated JSON schemas for capabilities auto-complete (gitignored in `src-tauri/.gitignore`).
- `build/` — frontend build output (referenced by `tauri.conf.json` as `frontendDist`).
