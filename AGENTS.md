# AGENTS.md — RustyNotes

Tauri 2.0 + SvelteKit (SPA) + TypeScript frontend / Rust backend. Obsidian-like note app.

## Commands

```bash
yarn tauri dev      # dev server + webview (runs yarn dev first)
yarn tauri build    # production build
yarn check          # type-check: svelte-kit sync && svelte-check
```

No linter, no formatter, no test runner configured. `yarn check` is the only verification.

## Architecture — where code lives

| Concern | Path |
|---------|------|
| Frontend (SvelteKit) | `src/` |
| Backend (Rust/Tauri) | `src-tauri/src/` |
| Tauri commands & builder | `src-tauri/src/lib.rs` |
| Entry point | `src-tauri/src/main.rs` |
| Design docs | `docs/ARCHITECTURE.md` |
| Task breakdown | `docs/TASKS.md` |

This is a single-crate, single-package repo (not a monorepo).

## Rust crate naming quirk (Windows)

The Cargo.toml lib is named `rustynotes_lib` because cargo on Windows barfs when lib and bin have the same name. `main.rs` calls `rustynotes_lib::run()`. If an agent writes `rustynotes::run()`, it will fail.

## SvelteKit in SPA mode

- Uses `@sveltejs/adapter-static` with `fallback: "index.html"`.
- SSR is disabled: `src/routes/+layout.ts` exports `export const ssr = false`.
- Vite dev port is **1420** (fixed, strict); HMR on 1421.
- Vite ignores `src-tauri/` in watch mode.

## Tauri 2 capabilities

Permissions are in `src-tauri/capabilities/default.json`. Currently `core:default` + `opener:default`. When adding FS ops, Tauri 2 requires explicit permissions in the capability file — not just code changes.

## Generated / git-ignored paths

- `.svelte-kit/` — generated by `svelte-kit sync` (part of `yarn check`). Required before type-checking.
- `src-tauri/target/` — Rust build artifacts.
- `src-tauri/gen/schemas` — generated JSON schemas for capabilities auto-complete.
